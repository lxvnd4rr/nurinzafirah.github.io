Welcome to My e-Portfolio

An elegant display of mobile development milestones, interactive design explorations, and sustainable education alignment.

Learnify is a mobile learning application developed to help students manage their learning materials more effectively through digital flashcards. The application allows users to create, organize, study, and share flashcard sets for various subjects, making revision more structured and engaging. Learnify also supports user authentication, cloud-based data synchronization, and local data storage to ensure a seamless learning experience. By providing an accessible and interactive platform for self-directed learning, Learnify aims to improve students' study habits and academic performance.

SDG 4

Quality Education

Learnify is cleanly aligned with United Nations Sustainable Development Goal (SDG) 4: Quality Education, which aims to ensure inclusive and equitable quality education and promote lifelong learning opportunities for all. The application supports this goal by:

  • Providing students with easy access to educational resources anytime and anywhere.
  • Encouraging active learning through flashcard-based revision techniques.
  • Helping learners improve knowledge retention and study efficiency.
  • Supporting self-paced and continuous learning beyond the classroom.
  • Enabling the sharing of learning materials among students, fostering collaborative learning.

Laboratory Exercises

Step-by-step technical foundations built using Jetpack Compose, exploring modern Android UI architecture.

Lab 1

UI Fundamentals

Exploration of foundational layout configurations in Jetpack Compose, setting structural foundations with essential layout components.

Reflection

Lab 1 helped me understand the fundamental layout components in Jetpack Compose, especially how Column, Row, and Box are used to arrange UI elements. At first, recreating an existing app layout was challenging, but it improved my understanding of screen structure and UI design. This lab provided a strong foundation for developing more complex mobile applications in later labs and projects.

Lab 2

User Interaction

Deep dive into building dynamic interfaces and orchestrating responses based on explicit runtime user events.

Reflection

Lab 2 helped me understand how state management works in Jetpack Compose using remember and mutableStateOf. At first, it was challenging to understand how user input could automatically update the UI, but through practice I learned how recomposition makes applications interactive and responsive. This lab improved my understanding of creating dynamic user interfaces and prepared me for developing more advanced features in future projects.

Lab 3

Material Design 3

Polishing application aesthetics according to cohesive system design rules, colors, typography, and clean layout patterns.

Reflection

lab 3 helped me understand how Material Design improves the overall appearance and user experience of a mobile application. I learned how to apply custom themes, colors, typography, and cards to create a more professional interface. The expandable card animation was slightly challenging at first, but it helped me understand how animations can make an app more interactive and engaging.

Lab 4

Navigation & ViewModels

Structuring complex application flows via structured screen routing mechanisms and decoupled architectural view data pipelines.

Reflection

lab 4 helped me understand how to implement Jetpack Compose Navigation to connect multiple screens in Learnify, making the app flow more structured and realistic like a real application. I also learned how ViewModel helps to store and manage shared data across screens, so user input and flashcard information remain consistent even when navigating between pages. Overall, this lab improved my understanding of multi-screen architecture and state management in Android development.

Lab 5

Local Persistence via Room

Integrating persistent data layers using structured SQLite abstractions within standard Android architectures.

Reflection

Lab 5 helped me understand how to implement Room Database as a persistent storage solution in Android, allowing data to remain saved even after the app is closed. I also understood how the MVVM architecture works in a complete flow using Entity, DAO, Repository, and ViewModel to manage and separate data logic properly. Overall, this lab improved my understanding of real-world mobile app architecture and how data persistence is handled in professional Android applications.
